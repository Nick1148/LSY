## 테스트 1: 나의 뇌 속 브라우저 탭

### 컨셉
> "당신의 뇌에는 지금 탭이 몇 개 열려있을까?"

브라우저 탭 메타포로 사고 패턴을 진단. 결과를 실제 브라우저 탭 UI로 보여줌.

### 질문 (5개)
1. **일요일 밤, 내일 출근인데 갑자기 드는 생각은?**
   - A) 내일 뭐 입지... 아 그리고 점심 뭐 먹지 (생활형 멀티탭)
   - B) 갑자기 3년 뒤 내 모습이 궁금해진다 (미래 시뮬레이션형)
   - C) 아까 본 릴스가 자꾸 생각남 (콘텐츠 중독형)
   - D) 별 생각 없이 천장 봄 (절전 모드형)

2. **친구가 고민 상담을 요청했다. 나의 반응은?**
   - A) 들으면서 동시에 해결책 3개를 구상 중 (병렬 처리형)
   - B) 공감하면서도 "근데 나도 비슷한 게 있는데..." (탭 전환형)
   - C) 100% 몰입해서 듣고 있음 (단일 탭 집중형)
   - D) 듣다가 갑자기 다른 생각이 떠오름 (팝업 탭형)

3. **샤워할 때 주로 하는 생각은?**
   - A) 오늘 있었던 일 복기 + 내일 계획 + 주말 약속 동시 처리
   - B) 갑자기 초등학교 때 창피했던 기억이 소환됨
   - C) 무의식적으로 노래 부르거나 아무 생각 없음
   - D) 방금 본 유튜브 영상 내용을 머릿속으로 정리

4. **잠들기 직전, 머릿속 상태는?**
   - A) 생각이 꼬리에 꼬리를 물어서 잠이 안 옴
   - B) "아 내일 그거 해야 하는데" 하나가 계속 맴돔
   - C) 이불 덮으면 3분 안에 잠듦
   - D) ASMR이나 영상 틀어놔야 잠듦 (백그라운드 탭 필수)

5. **카페에서 공부/작업 중일 때 나는?**
   - A) 옆 테이블 대화 + 음악 + 작업 동시 처리 가능
   - B) 집중하다가 갑자기 핸드폰 들여다봄 (새 탭 자동 열림)
   - C) 이어폰 끼고 완전 몰입 모드
   - D) 30분 작업 → 인스타 10분 → 작업 반복 (탭 순환형)

### 결과 유형 (6가지)
각 결과는 **실제 브라우저 탭 바 UI**로 보여줌

| 유형 | 탭 개수 | 설명 |
|------|---------|------|
| 🔥 과부하 브라우저 | 47개+ | "RAM 부족 경고! 당신의 뇌는 항상 과부하 상태" |
| 🎯 효율형 브라우저 | 3~5개 | "필요한 탭만 딱딱 열어두는 미니멀리스트" |
| 🌀 탭 순환러 | 15~20개 | "열어두고 안 보는 탭 전문가. 닫기는 불안해서 못 닫음" |
| 💤 절전 모드 | 1~2개 | "뇌 배터리 절약의 달인. 멀티태스킹? 그게 뭔데?" |
| 🎪 팝업 탭 전문가 | 탭 수 랜덤 | "예고 없이 새 탭이 열리는 예측불가 뇌" |
| 🔄 백그라운드 러너 | 8~12개 | "겉으론 조용하지만 뒤에서 조용히 돌아가는 탭 다수" |

### 결과 카드 디자인
- 실제 Chrome/Safari 탭 바 모양
- 각 탭에 재미있는 제목 표시 (예: "쓸데없는 고민.exe", "내일 점심 뭐먹지.hwp")
- 하단에 "RAM 사용량: 87%" 같은 유머 요소
- 공유 버튼 (카카오톡, 인스타 스토리, 링크 복사)

## 테스트 2: 나의 사회적 배터리 잔량

### 컨셉
> "당신의 사회적 배터리는 지금 몇 %?"

사회적 에너지를 스마트폰 배터리에 비유. 충전 방법까지 안내.

### 질문 (5개)
1. **금요일 퇴근 후, 친구에게서 "오늘 술 한잔?" 연락이 왔다.**
   - A) 오 좋아! 어디로 갈까? 🔋+20%
   - B) 누구 오는데? 멤버 보고 결정 🔋+5%
   - C) 아... 오늘은 좀... (하지만 거절 못 함) 🔋-10%
   - D) 읽씹 후 넷플릭스 켬 🔋-20%

2. **단톡방에 30개의 안 읽은 메시지가 있다.**
   - A) 바로 확인하고 대화에 참여 🔋+10%
   - B) 슬쩍 읽고 중요한 것만 답장 🔋+0%
   - C) 나중에 읽어야지... (3일 후에도 안 읽음) 🔋-15%
   - D) 단톡방 알림 이미 꺼놨음 🔋-25%

3. **엘리베이터에서 아는 사람을 마주쳤다.**
   - A) 반갑게 인사하고 근황 토크 🔋+15%
   - B) 가볍게 목인사 + 미소 🔋+5%
   - C) 핸드폰 보는 척 🔋-10%
   - D) 계단으로 간다 🔋-30%

4. **회사/학교에서 팀 프로젝트를 하게 됐다.**
   - A) 리더 자청! 역할 분배까지 함 🔋+20%
   - B) 주어진 역할은 잘 해내는 팀원 🔋+5%
   - C) 혼자 할 수 있는 파트 선택 🔋-5%
   - D) 왜 굳이 팀으로... (내면의 비명) 🔋-20%

5. **오랜만에 만난 모임에서 새로운 사람을 소개받았다.**
   - A) 먼저 말 걸고 인스타 교환 🔋+15%
   - B) 상대가 말 걸면 반응 좋게 대화 🔋+5%
   - C) 옆에서 듣고만 있다가 슬쩍 빠짐 🔋-15%
   - D) 화장실 간다고 하고 5분간 숨쉬기 🔋-25%

### 결과 유형 (5단계)
배터리 UI로 표시

| 잔량 | 유형명 | 설명 |
|------|--------|------|
| 100~80% | ⚡ 완충 소셜킹 | "사람 만나는 게 충전! 약속 없는 주말이 더 불안한 타입" |
| 79~60% | 🔋 양호한 사교인 | "적당히 만나고 적당히 쉬는 밸런스형. 가끔 혼자 시간 필요" |
| 59~40% | 🪫 절전모드 사회인 | "사회생활은 하지만 집이 최고. 약속 취소되면 내심 기뻐함" |
| 39~20% | 🔴 저전력 경고 | "사회적 에너지 위험 수준! 충전 시급. 최소 3일 혼자 시간 필요" |
| 19~0% | 💀 방전 히키코모리 | "배터리 나감. 현재 사회와 연결 해제됨. 이불 속이 세상의 전부" |

### 결과 카드 디자인
- 스마트폰 배터리 아이콘 (잔량에 따라 색상 변화)
- "충전 방법 3가지" 추천 (유형별 맞춤)
- "이 배터리와 잘 맞는 사람" 궁합 표시
- 공유 버튼

## 기술 구현 계획

### 프로젝트 구조
```
viral-test/
├── index.html              # 메인 랜딩 (테스트 목록)
├── brain-tabs/
│   ├── index.html          # 뇌 속 브라우저 탭 테스트
│   ├── css/style.css
│   └── js/app.js
├── social-battery/
│   ├── index.html          # 사회적 배터리 테스트
│   ├── css/style.css
│   └── js/app.js
├── shared/
│   ├── css/common.css      # 공통 스타일 (버튼, 폰트 등)
│   ├── js/share.js         # 공유 기능 (카카오톡, 클립보드)
│   └── js/analytics.js     # 방문자 추적 (선택)
└── assets/
    ├── og-brain-tabs.png   # OG 이미지 (SNS 미리보기)
    └── og-social-battery.png
```

### 핵심 기능
1. **질문 UI**: 카드 스와이프 or 세로 스크롤 형태
2. **프로그레스 바**: 진행률 표시
3. **결과 계산**: 선택지별 점수 합산 → 유형 매칭
4. **결과 카드 생성**: Canvas API or HTML2Canvas로 이미지화
5. **공유 기능**:
   - 카카오톡 공유 (Kakao SDK)
   - 인스타그램 스토리 공유 (이미지 다운로드 → 스토리 업로드 유도)
   - URL 링크 복사
   - 트위터 공유
6. **OG 태그**: SNS에서 링크 공유 시 미리보기 최적화

### 디자인 방향
- **모바일 퍼스트** (90%+ 모바일 접속 예상)
- **다크 테마** + 네온 컬러 (트렌디한 느낌)
- **마이크로 인터랙션**: 선택 시 애니메이션, 결과 공개 시 연출
- **카드형 질문**: 한 화면에 질문 하나씩
- **타이포 중심**: 일러스트 없이도 임팩트 있는 텍스트 디자인

### 바이럴 장치
1. **결과 공유 CTA**: "친구한테 공유하고 비교해봐!" 강조
2. **다른 테스트 추천**: 결과 하단에 다른 테스트 유도
3. **궁합 기능**: "이 유형과 잘 맞는 사람은?" → 상대방도 테스트하게 유도
4. **OG 이미지 최적화**: 공유 시 클릭 유도하는 썸네일

## 개발 순서 (우선순위)

### Phase 1: 뇌 속 브라우저 탭 테스트 (먼저 개발)
1. 프로젝트 기본 구조 세팅
2. 공통 컴포넌트 (질문 UI, 프로그레스 바, 결과 카드)
3. 질문 흐름 + 결과 계산 로직
4. 결과 페이지 + 브라우저 탭 UI
5. 공유 기능 (카카오톡, 링크복사)
6. OG 태그 + 메타 데이터
7. 모바일 최적화 + 애니메이션 폴리싱

### Phase 2: 사회적 배터리 테스트
1. 배터리 테스트 질문/결과 데이터 세팅
2. 배터리 UI 컴포넌트 개발
3. 결과 페이지 + 충전 방법 추천
4. 공유 기능 연동

### Phase 3: 메인 랜딩 + 최적화
1. 메인 페이지 (테스트 목록)
2. Google Analytics 연동
3. 성능 최적화
4. SEO 기본 설정

## 참고: Google Cloud API 활용 가능 부분
- **Vision API**: 추후 사진 기반 테스트 추가 시 활용
- **Translation API**: 다국어 지원 시
- **Firebase Hosting**: GitHub Pages 대안 (커스텀 도메인 SSL 무료)
- **Firebase Analytics**: 상세 사용자 분석
- **Firestore**: 결과 통계 저장 (예: "전체 참여자 중 당신은 상위 23%")

현 단계에서는 Google Cloud 없이 순수 프론트엔드로 충분합니다.
